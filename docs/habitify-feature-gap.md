# Habitify 功能对照与微信小程序差距分析

## 1. 结论

我们当前设计的微信小程序已经覆盖 Habitify 的核心闭环：创建习惯、今日清单、提醒、打卡记录、时长 / 次数统计、连续天数、日历和趋势图。对「念诵经文、深蹲、学习英语」这类个人习惯，MVP 是够用的。

但它还不是完整的 Habitify。Habitify 已经是成熟多端产品，公开资料中包含 Focus Timer、跨设备同步、Apple Watch、Apple Health、RescueTime、Zapier、IFTTT、API、桌面端 / 移动端协同、自定义时间段、Checklist、Notes 等能力。微信小程序 MVP 需要先做核心功能，其他能力分阶段补齐。

## 2. 功能对照表

| Habitify 能力 | 我们的小程序 MVP 是否覆盖 | 当前设计位置 | 差距 / 建议 |
| --- | --- | --- | --- |
| 创建自定义习惯 | 已覆盖 | PRD 的习惯创建 | 支持名称、图标、颜色、分组、目标类型、频率、提醒和目标值。 |
| 今日习惯清单 | 已覆盖 | PRD 的今日页 | MVP 已按时间段或分组展示，后续可加入拖拽排序。 |
| 完成 / 跳过 / 失败记录 | 已覆盖 | PRD 的记录页 | 已设计状态字段和备注。 |
| 计数型目标 | 已覆盖 | PRD 的目标类型 | 适合深蹲、喝水、阅读页数等。 |
| 时长型目标 | 已覆盖 | PRD 的目标类型 | 适合念诵经文、学习英语、冥想等。 |
| 完成率、连续天数、总投入 | 已覆盖 | PRD 统计页、技术方案统计算法 | MVP 已能回答“念诵了多久、坚持了几天”。 |
| 日历视图 / 热力图 | 已覆盖 | PRD 统计页、路线图第 4 周 | 第 4 周做增强更合适，第一版可先做简单日历。 |
| 趋势图 | 已覆盖 | PRD 统计页、路线图第 4 周 | 近 7 天 / 近 30 天优先。 |
| 提醒通知 | 部分覆盖 | PRD 温和提醒、技术方案提醒实现 | 微信小程序依赖订阅消息，不能完全等同原生 App 的本地通知和闹钟。 |
| Focus Timer / Stopwatch | 暂未进入 MVP | PRD 后续增强 | 建议 P2 做，念诵经文和学习英语很需要。 |
| Notes / Journal | 部分覆盖 | PRD 记录页备注 | MVP 只有备注，尚未做完整日记视图和按习惯回顾。 |
| Checklist / 子步骤 | 未覆盖 | 暂无 | 适合“早课流程”“晨间流程”，建议 P2。 |
| 自定义时间段 | 部分覆盖 | PRD 今日页时间段 | MVP 有上午 / 下午 / 晚上 / 任意时间，尚未支持用户自定义“夜班”“深度学习”等时间块。 |
| 坏习惯 / 限制目标 | 部分覆盖 | PRD 目标类型 | 已有“限制型”概念，但路线图未列为 P0。建议 P2。 |
| 跨设备同步 | 部分覆盖 | README MVP 范围、技术方案可扩展方向 | 微信小程序天然可在微信内跨手机使用，但不等于 iOS / Android / Web / Mac 全平台同步。 |
| Apple Watch | 不覆盖 | 无 | 微信小程序不适合直接做 Apple Watch，除非后续开发原生 iOS App。 |
| Apple Health / 自动药物记录 | 不覆盖 | 无 | 微信小程序能力受限，后续若做原生 App 再考虑。 |
| RescueTime 自动同步 | 不覆盖 | 无 | 可作为高级集成，但不建议 MVP 做。 |
| Zapier / IFTTT / API | 不覆盖 | 无 | 适合成熟后给重度用户或企业用户。 |
| Widget / 桌面小组件 | 不覆盖 | 无 | 微信小程序不能提供完整系统级桌面小组件体验。 |
| 数据导出 | 已规划 | PRD 后续增强、路线图第 4 周 | 建议 CSV 先做。 |
| 好友监督 / 社交 | 未覆盖 MVP | PRD 后续增强 | Habitify 本身也偏核心习惯而非强游戏化，社交不是第一优先级。 |
| AI 复盘 | 未覆盖 MVP | PRD 后续增强、路线图第 5 周以后 | 可做差异化，但不要抢核心功能优先级。 |

## 3. 对你的三个例子是否够用

### 3.1 念诵经文

MVP 可以满足：

- 每天 06:30 提醒。
- 点“开始计时”。
- 完成后记录 35 分钟、备注“念诵 2 遍”。
- 统计总念诵时长、连续天数、近 30 天趋势。

还缺：

- 更顺手的 Focus Timer。
- 按经文 / 功课拆分子步骤，例如“心经、金刚经、回向”。
- 念诵日记视图。

### 3.2 深蹲

MVP 可以满足：

- 每周 5 天提醒。
- 每天记录 50 个或实际完成个数。
- 查看周总次数、近 30 天趋势和完成率。

还缺：

- 与运动手表或健康数据自动同步。
- 多组训练记录，例如 20 + 20 + 10。

### 3.3 学习英语

MVP 可以满足：

- 每天 21:00 提醒。
- 记录学习 30 分钟 / 45 分钟。
- 备注学习内容，例如“背单词 50 个、听力 20 分钟”。
- 查看总学习时长和连续天数。

还缺：

- 番茄钟 / 专注计时器。
- 子任务 Checklist，例如“单词、听力、阅读、口语”。
- 自动从第三方学习 App 同步数据。

## 4. 推荐补齐顺序

### P0：一定要先做

- 创建习惯。
- 今日页。
- 打卡记录。
- 计数 / 时长目标。
- 基础统计。
- 微信订阅消息提醒。

### P1：让产品像 Habitify

- 日历热力图。
- 近 7 天 / 近 30 天趋势图。
- 当前连续天数和最长连续天数。
- 数据导出 CSV。
- 更细的提醒设置。

### P2：让念诵、学习场景更好用

- Focus Timer / Stopwatch。
- Checklist 子步骤。
- Notes / Journal 独立视图。
- 自定义时间段。
- 坏习惯限制目标。

### P3：成熟产品能力

- Web / App 多端。
- Apple Health / 手表数据。
- RescueTime、Zapier、IFTTT 等自动化集成。
- API。
- AI 周报和个性化建议。

## 5. 取舍建议

不要一开始就追求完整复制 Habitify。微信小程序的优势是轻、快、容易打开；弱点是系统通知、健康数据、手表和桌面小组件能力不如原生 App。

所以第一版应该做到：

```text
今天该做什么 → 到点提醒 → 快速记录 → 看见自己坚持了多久
```

只要这条链路顺，就已经能解决你的核心需求。等有真实用户持续记录后，再补 Focus Timer、Checklist、日记和高级集成。
