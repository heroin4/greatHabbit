## 3. 订阅消息模板设计

### 3.1 习惯待完成提醒

用途：到点提醒用户完成某个习惯。

推荐字段：

| 字段 | 含义 | 示例 |
| --- | --- | --- |
| 习惯名称 | 用户要完成的习惯 | 念诵经文 |
| 提醒时间 | 用户设置的提醒时间 | 06:30 |
| 目标 | 今日目标 | 30 分钟 |
| 温和提示 | 不制造焦虑的提示语 | 愿你今天也稳稳完成 |

推荐文案：

```text
你设置的「念诵经文」时间到了
目标：30 分钟
愿你今天也稳稳完成
```

### 3.2 每日复盘提醒

用途：晚上提醒用户查看今日完成情况。

推荐字段：

| 字段 | 含义 | 示例 |
| --- | --- | --- |
| 复盘事项 | 固定标题 | 今日习惯复盘 |
| 复盘时间 | 触发时间 | 21:30 |
| 提示 | 说明用户可查看的内容 | 看看今天坚持了哪些习惯 |

推荐文案：

```text
今日习惯复盘时间到了
看看今天坚持了哪些习惯
```

## 4. 当前实现边界

现在已经实现：

- 小程序启动时初始化云开发。
- 订阅消息模板配置文件。
- 前端订阅授权入口。
- `sendReminder` 云函数骨架。
- 根据习惯生成订阅消息 data 的工具函数。

还需要你申请 AppID 后完成：

1. 在微信公众平台申请订阅消息模板。
2. 把模板 ID 填入 `miniprogram/config/reminder-templates.js`。
3. 在云开发控制台创建环境，并把环境 ID 填入 `miniprogram/config/cloud.js`。
4. 上传并部署 `cloudfunctions/sendReminder`。
5. 配置定时触发器，按分钟或 5 分钟扫描待提醒习惯。

## 5. 提醒语气原则

提醒不要像任务惩罚，而要像温和陪伴：

- 少用“必须、立刻、失败”等词。
- 多用“到时间了、稳稳完成、做一点也很好”。
- 已完成的习惯当天不再提醒。
- 晚间复盘只提醒一次，不连续打扰。
