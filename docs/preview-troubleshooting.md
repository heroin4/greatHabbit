# 微信开发者工具预览没有变化排查

如果你已经 fetch / pull 了 `main`，但微信开发者工具里看起来没有变化，通常不是代码没改，而是下面几类原因。

## 1. 确认拉到的是最新 main

在项目根目录执行：

```bash
git checkout main
git pull origin main
git log --oneline -5
```

最新代码里应该能看到视觉优化相关提交，或者至少看到 `miniprogram/config/app-meta.js` 这个文件。

## 2. 确认导入的是仓库根目录

微信开发者工具导入项目时，目录应该选仓库根目录，而不是 `miniprogram/` 子目录。项目根目录里有 `project.config.json`，其中配置了：

```json
{
  "miniprogramRoot": "miniprogram/"
}
```

## 3. 清理开发者工具缓存

微信开发者工具里可以依次尝试：

1. 点击「编译」。
2. 点击「清缓存」→「清除数据缓存」。
3. 点击「清缓存」→「清除文件缓存」。
4. 关闭项目后重新导入。

## 4. 为什么默认习惯可能没变化？

默认习惯只在第一次启动时写入本地缓存。如果你之前已经打开过旧版本，小程序本地缓存里会保留旧数据，新的默认模板不会自动覆盖。

现在「我的」页新增了「重置演示数据」按钮，可恢复默认习惯模板并清空本地打卡记录。

## 5. 如何一眼确认是不是新版 UI？

新版「今日」页顶部会显示：

```text
LifeTech v0.2
```

「我的」页也会显示同样的视觉版本徽标。如果你看不到这个版本号，说明开发者工具打开的不是最新代码，或者缓存没有清理。
