# MVP 迭代路线图

## 第 0 周：产品原型

- 输出低保真原型：今日页、新建习惯、记录弹窗、统计页。
- 确认目标类型：完成、计数、时长、限制。
- 确认首批模板：念诵经文、深蹲、学习英语、饮水、早睡。

## 第 1-2 周：可用闭环

- 微信登录与用户初始化。
- 新建 / 编辑 / 归档习惯。
- 今日习惯列表。
- 完成型、计数型、时长型记录。
- 基础统计：完成率、连续天数、总次数 / 总时长。

验收标准：用户可以创建「念诵经文 30 分钟」并连续记录 7 天，统计页能展示总念诵时长。

## 第 3 周：提醒与体验优化

- 接入订阅消息。
- 支持提醒时间设置。
- 今日已完成后不再提醒。
- 增加空状态、完成动效、快捷模板。

验收标准：用户能为「学习英语」设置每天 21:00 提醒，并在小程序内完成记录。

## 第 4 周：统计增强

- 日历热力图。
- 近 7 天 / 近 30 天趋势。
- 全局统计总览。
- 数据导出 CSV。

验收标准：用户能查看「深蹲」近 30 天每天完成次数和总次数。

## 第 5 周以后：增长与高级功能

- 专注计时器。
- 坏习惯限制。
- 习惯堆叠提醒。
- 好友监督。
- AI 周报与调整建议。

## 推荐优先级

| 优先级 | 功能 | 原因 |
| --- | --- | --- |
| P0 | 创建习惯、今日打卡、日志、基础统计 | 构成产品主闭环 |
| P1 | 订阅消息、日历热力图、趋势图 | 提升坚持和复盘价值 |
| P2 | 专注计时器、模板、数据导出 | 提升具体场景体验 |
| P3 | 好友、AI、跨端 | 增长与高级付费能力 |
