# 习惯奖励系统初步方案

## 1. 为什么做奖励

统计页不应该只展示数字，也应该给用户“我被看见了”的正反馈。奖励不建议做得太游戏化，而是像 Habitify 的数据洞察一样，帮助用户确认自己的坚持正在累积。

## 2. MVP 奖励规则建议

| 完成天数 | 奖励 | 设计语气 |
| --- | --- | --- |
| 3 天 | 🌱 三日萌芽 | 习惯开始发芽 |
| 7 天 | 🌟 七日稳定星 | 节奏开始稳定 |
| 30 天 | 🥇 月度守护者 | 已经完成一个月 |
| 100 天 | 🏆 百日大师 | 坚持成为生活方式 |

## 3. 当前代码里的预埋实现

- `miniprogram/utils/stats.js` 会根据每个习惯的完成天数返回对应奖励。
- 统计页会展示“奖励设想”卡片和每个习惯当前点亮的奖励。
- 现在只是轻量展示，后续可以做奖励详情页、徽章墙、分享海报。

## 4. 后续讨论点

1. 奖励按“累计完成天数”还是“连续完成天数”？
2. 是否允许用户自定义奖励，比如坚持 7 天奖励自己看一场电影？
3. 宗教功课类习惯是否应该使用更庄重、少游戏化的奖励文案？
4. 是否要做每周复盘报告，把奖励和鼓励话术一起展示？
